Dossier d'étude - SAÉ S1.03

Ce dossier présente les choix techniques et les justifications pédagogiques adoptés pour la réalisation du projet SAÉ S1.03 : Virtualisation et déploiement web. Chaque décision a été prise en fonction des objectifs d'apprentissage pour des étudiants en première année d'informatique.

Contexte et objectifs pédagogiques

La SAÉ S1.03 nous a permis de découvrir et de pratiquer des notions essentielles en informatique : la virtualisation, l’administration système et la mise en ligne d’un site web. L’objectif principal était d’apprendre à utiliser un environnement Linux, à installer des services réseau et à comprendre le déploiement d’un site web. Cette SAÉ nous a également aidé à développer des compétences importantes comme la résolution de problèmes, l’organisation du travail et la rédaction technique.

Choix de la virtualisation : VirtualBox et Ubuntu 22.04 LTS

Pour réaliser le projet, nous avons utilisé VirtualBox comme hyperviseur. C’est un logiciel gratuit, simple à prendre en main et sûr, ce qui est parfait pour des étudiants débutants. Son interface graphique nous a permis de manipuler facilement la machine virtuelle sans risque pour l’ordinateur principal.

Nous avons choisi Ubuntu 22.04 LTS comme système d’exploitation invité pour sa stabilité et son support long terme. Cette distribution est adaptée aux débutants car elle est bien documentée et propose tous les outils nécessaires pour un environnement de développement. Elle est également représentative des systèmes utilisés sur des serveurs Linux réels.

La configuration de la machine virtuelle (4GB RAM, 40GB disque dynamique, réseau NAT) nous a permis d’avoir un environnement fonctionnel tout en restant compatible avec les ressources disponibles sur la machine hôte.

Installation et configuration du serveur web Apache

Nous avons installé Apache comme serveur web pour héberger notre site. L’installation s’effectue avec la commande : sudo apt update && sudo apt install apache2. Apache est un serveur web fiable et largement utilisé, ce qui nous a permis de nous familiariser avec des outils professionnels dès la première année.

Nous avons conservé la configuration par défaut d’Apache, suffisante pour ce projet. Les fichiers du site ont été placés dans /var/www/html/, et le site est accessible via http://localhost/. Cette configuration simple nous a permis de nous concentrer sur la compréhension des concepts plutôt que sur les détails avancés du serveur.

Le déploiement des fichiers via WebStorm nous a permis de comprendre comment synchroniser le code local avec la machine virtuelle et de pratiquer la gestion des fichiers sur un serveur distant.

Développement du site web : Bootstrap 5 et approche modulaire

Pour créer le site, nous avons utilisé Bootstrap 5, qui permet de construire un site responsive facilement. Cela nous a aidé à nous concentrer sur la structure et le contenu plutôt que sur le CSS complexe. Les couleurs et les polices choisies (violet, bleu, jaune ; Montserrat et Fira Code) rendent le site agréable à lire tout en conservant une présentation claire.

Le site est organisé en pages séparées pour chaque partie du projet, ce qui facilite la lecture et la maintenance. Cette approche montre l’importance de structurer notre travail et de respecter les bonnes pratiques dès le début.

Cohérence pédagogique et compétences acquises

L’ensemble des outils choisis nous a permis de découvrir un environnement complet : VirtualBox pour pratiquer la virtualisation, Ubuntu pour apprendre à utiliser Linux, Apache pour comprendre le serveur web et Bootstrap pour construire un site responsive. Chaque outil a été introduit progressivement pour faciliter l’apprentissage.

Ce projet nous a permis de développer à la fois des compétences techniques (installation, configuration, déploiement) et professionnelles (organisation, documentation, analyse des contraintes techniques).

Commentaires sur la vidéo pédagogique

The video presentation aims to explain the fundamental principles of virtualization to a technical audience. The oral delivery was focused on three main pillars : Definition : Explaining how a hypervisor allows multiple Operating Systems to run simultaneously on a single physical machine. Advantages : Highlighting cost reduction, environment isolation for developers, and the ability to test different OS (like Ubuntu) without risking the host system. Drawbacks :Discussing resource overhead (CPU/RAM consumption) and poten- tial latency compared to native installations. Cette séquence nous a permis de mobiliser le vocabulaire technique lié à l’architecture système (host, guest, hypervisor, overhead, scalability) tout en travaillant sur la clarté de l’explication orale. La mise en forme "vidéo YouTube" nous a imposé une rigueur de synthèse pour rendre le sujet accessible et dynamique. Il nous a permis de mieux comprendre le fonctionnement d’une machine virtuelle.
